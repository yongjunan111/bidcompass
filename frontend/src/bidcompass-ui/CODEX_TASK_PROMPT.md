# Codex 작업 지시문

아래 지시를 그대로 Codex에게 전달하면 됩니다.

---

이 저장소에 포함된 `BidCompass Unified UI Kit`를 기준으로 실제 서비스 프론트를 연결해 주세요.

## 목표
현재 들어있는 UI 컴포넌트와 페이지 디자인을 **재디자인하지 말고 그대로 유지**한 상태에서,
기존 프로젝트 구조에 맞게 라우팅과 데이터 연결만 수행해 주세요.

## 우선 작업
1. repo 구조를 먼저 파악해 주세요.
2. 이미 있는 프론트 구조(Next.js, React Router, Vite 등)에 맞춰 아래 페이지를 연결해 주세요.
   - 대시보드
   - 공고 조회
   - 추천 결과
   - 가격 계산
   - AI 리포트
   - 히스토리
   - 설정
3. `AppShell`과 `bidcompass-ui.css`를 공통 기반으로 사용해 주세요.
4. `data/mock.ts`는 실제 API 응답으로 교체하되, 가능하면 **adapter layer**를 만들어 UI props 형식은 유지해 주세요.
5. 추천 결과 페이지는 핵심 화면이므로, 현재 카드 구조와 우측 AI 요약 패널을 바꾸지 말아 주세요.
6. 구어체 문구로 바꾸지 말고 현재 문어체 톤을 유지해 주세요.
7. 기존 프로젝트 스타일 시스템이 있더라도 이 화면군은 토큰과 spacing을 최대한 유지해 주세요.
8. 작업 후에는 다음 내용을 정리해 주세요.
   - 변경한 파일 목록
   - 연결한 라우트 목록
   - mock → real API 매핑 방식
   - 남은 TODO
   - 실행/빌드/테스트 결과

## 구현 원칙
- 디자인 변경보다 연결 작업 우선
- 불필요한 UI 라이브러리 추가 금지
- 상태는 최소한 `empty / loading / success / error` 4개를 고려
- 타입이 다르면 UI를 바꾸지 말고 adapter에서 해결
- CSS class naming은 가급적 유지
- 새 컴포넌트가 필요하면 기존 클래스 규칙에 맞춰 추가

## 추천 순서
1. 공통 shell 이식
2. 공고 조회 + 추천 결과 연결
3. 가격 계산 연결
4. AI 리포트 연결
5. 히스토리/설정 연결
6. 상태 처리 정리
7. 테스트 및 빌드 확인

## 중요
- layout 구조, 색상 톤, 카드 스타일, 제목 hierarchy는 유지해 주세요.
- “더 예쁘게” 바꾸려 하지 말고, **지금 승인된 디자인을 제품에 옮기는 작업**으로 이해해 주세요.
