# BidCompass Unified Design System

## 1. 목적
추천 결과 페이지에서 마음에 들었던 디자인을 기준으로 전체 제품을 한 톤으로 통일한다.
이 문서는 Codex가 화면을 연결하거나 확장할 때 기준이 되는 **UI 규격서**다.

## 2. 제품 구조
### 핵심 흐름
1. 공고 조회
2. 추천 결과
3. 가격 계산 검증
4. AI 리포트 생성
5. 히스토리 재사용

### 보조 화면
- 대시보드
- 설정

## 3. 레이아웃 규칙
- 상단: 브라우저형 바 + 현재 경로
- 좌측: 사이드바 네비게이션
- 중앙: 주 판단 영역
- 우측: 보조 정보, 액션, 상태
- 레이아웃 비율 기본값  
  `252px / fluid / 344px`

## 5. 컴포넌트 규칙
### AppShell
모든 페이지는 같은 shell을 사용한다.
- 브라우저 바
- 좌측 메뉴
- 메인 제목/부제
- 우측 aside

### Panel
가장 기본 카드.
- 패딩 18px
- 라운드 18px
- 내부 문장 길이는 짧게 유지

### MetricStrip
숫자 요약 카드.
- 대시보드, 리포트 상단에서 사용
- 4칸 그리드 기본

### StatusBanner
로딩/완료/주의/오류 상태용
- 색상과 텍스트를 같이 쓴다
- 빨강/초록만으로 구분하지 않는다

### StrategyGroup
추천 결과 페이지 전용 핵심 컴포넌트.
- 안정형 / 균형형 / 적극형
- 선택 상태가 명확해야 한다
- 클릭 시 다른 값들이 함께 갱신된다는 구조를 유지

## 6. 카피 규칙
- 구어체 금지
- 짧은 문장 사용
- “설명”보다 “상태 + 수치 + 결론” 우선
- 버튼 라벨은 명사보다 동사형 우선  
  예: `분석 시작`, `PDF 생성`, `추천값 복사`

## 7. 페이지별 목적
### 대시보드
운영 상태를 빠르게 보는 화면

### 공고 조회
검색과 최근 조회를 처리하는 입력 화면

### 추천 결과
제품의 핵심 화면
- 추천 3안
- 가격 판단
- AI 해설
- 유사 공고 데이터

### 가격 계산
숫자 검증용 상세 화면

### AI 리포트
의사결정 문장과 근거 정리 화면

### 히스토리
이전 분석 재사용 화면

### 설정
팀 기본값과 출력 정책 관리 화면

## 8. 연결 원칙
- 기존 서비스에 붙일 때는 **디자인을 다시 짜지 않는다**
- 데이터 모델이 다르면 API adapter에서만 변환한다
- layout, spacing, token, panel 스타일은 그대로 유지한다
- 새 페이지가 생겨도 AppShell 구조를 유지한다

## 9. 확장 우선순위
1. 로딩 상태
2. 에러 상태
3. 빈 상태
4. 모바일 세부 튜닝
5. 실제 차트/표 연결
